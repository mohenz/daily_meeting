# UI 개편 및 개발 이행 계획 (UI Revamp Implementation Plan)

## 1. 개요
`daily_meeting_memo_app_spec.md`의 기능 요구사항을 바탕으로, 사용자 경험(UX)과 시각적 완성도(Aesthetics)를 극대화하는 UI 개편을 진행합니다. 단순한 기능 구현을 넘어, '운영 도구'로서의 신뢰감과 '협업 도구'로서의 생동감을 제공하는 프리미엄 디자인을 적용합니다.

## 2. 디자인 전략 (Visual Strategy)
- **Concept**: Executive Dark & Vibrant Mint (심도 있는 다크 톤과 선명한 포인트 컬러의 조화)
- **Typography**: `Inter` (Sans-serif) 기반의 명확한 정보 위계 설정
- **Layout**: 사이드바를 통한 타임라인 접근과 메인 워크스페이스의 카드 그리드 시스템
- **Interactions**: 부드러운 호버 효과 및 탭 전환 애니메이션

## 3. 화면별 상세 고도화 계획

### 3.1 가입 및 세션 화면 (Auth View)
- **현상**: 단순 폼 구조
- **개선**: 중앙 집중형 카드 레이아웃, 익명 세션 준비 상태의 시각적 피드백 강화, "이름+이메일" 입력의 간결함 강조

### 3.2 날짜 목록 사이드바 (Sidebar View)
- **현상**: 단순 리스트
- **개선**: 날짜별 요약 지표(Scrum/Wrap-Up 채워짐 정도)를 미니 게이지 또는 도트 아이콘으로 시각화, 선택된 날짜의 강조 효과 극대화

### 3.3 데일리 보드 헤더 (Workspace Header)
- **현상**: 텍스트 중심
- **개선**: 현재 날짜와 전체 참여 현황을 요약하는 대시보드형 헤더, 관리자용 미팅 생성 버튼의 시인성 확보

### 3.4 작업자 현황 카드 워크스페이스 (Workspace Body)
- **현상**: 테이블 또는 단순 리스트
- **개선**: 작업자별 상세 카드 시스템 적용. 4대 항목(진행완료, 진행중, 예정, 이슈)을 구분하는 내부 그리드와 본인/관리자 여부에 따른 액션 핸들 배치

## 4. 기술 이행 단계

1. **[CSS] 디자인 시스템 갱신**: `style.css`의 변수와 공통 컴포넌트 스타일을 프리미엄 톤으로 재정의
2. **[HTML/JS] 구조 개선**: `index.html` 및 각 View 모듈(`js/views/`)의 마크업 구조를 개편된 디자인에 맞게 수정
3. **[JS] 인터랙션 강화**: 데이터 업데이트 시의 시각적 피드백(Skeleton UI 또는 로딩 상태) 보완
4. **[Refinement] 최종 세니티 체크**: 전체 UI 정렬, 반응형 동작, 권한별 버튼 노출 여부 확인
